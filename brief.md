## 1. Цель сайта

### Главная цель
Конвертировать посетителя в **квалифицированную заявку** — человека, который написал в Telegram или заполнил форму с описанием своей задачи.

### Вторичные цели
- Сформировать доверие за счёт конкретных кейсов с результатами (сейчас главный gap)
- Отсеять нецелевых клиентов (секция "Это для вас / не для вас")
- Демонстрировать мастерство не только словами, но и качеством самого сайта
- Дать исчерпывающие ответы на вопросы, которые обычно задают на первой встрече

### Что сайт НЕ должен делать
- Быть универсальным портфолио для всех (фокус — веб + приложения + AI-агенты)
- Звучать как агентство или корпорация
- Скрывать информацию о ценах, сроках, процессе (это отталкивает клиентов)

## 2. Целевая аудитория

### Первичная аудитория
**Малый и средний бизнес с конкретной задачей**
- Стартапы, которым нужен MVP как можно быстрее
- Малый бизнес, который хочет автоматизировать рутину
- Предприниматели, у которых есть идея, но нет технической команды

**Боли:**
- Нормальный разработчик стоит дорого и занят на 3 месяца вперёд
- Агентства берут большие деньги и делают по шаблону с 40-страничным брифом
- "Не знаю, что мне нужно — сайт, бот или что-то другое"
- "Хочу попробовать AI-автоматизацию, но непонятно с чего начать"
- **Скрытая боль #1:** Время — пока ищут подрядчика, конкурент уже запустился
- **Скрытая боль #2:** "Подойдёт ли мне это?" — страх нерелевантности решения

### Вторичная аудитория
- Компании с амбициями роста, которым нужен дополнительный продукт или инструмент
- Команды, которые хотят добавить AI-функциональность в существующий продукт

### Портрет идеального клиента
Предприниматель или руководитель, 28–45 лет, думает задачами и результатами, а не технологиями. Готов платить адекватную цену за скорость и предсказуемость. Ценит прямой контакт без менеджеров.

## 3. Позиционирование и главный оффер

### Ключевое дифференцирование

| Обычный разработчик / агентство | Светлана — AI Builder |
|---|---|
| Пропадает на месяц | Первый результат через 24–48 часов |
| Нужен подробный бриф | Разбирается в бизнес-задаче сама |
| Делает по ТЗ, не думает о результате | Советует, что реально нужно |
| Дорого и долго | Быстро и по делу |
| Продаёт инструмент | Продаёт скорость и результат |

**Ключевое отличие:** Думает как предприниматель, строит как инженер — редкое сочетание.

### Рабочие формулировки оффера (выбрать одну для hero)

> **Вариант A (на скорость):** "Ваш сайт, приложение или AI-агент — за 3–7 дней"

> **Вариант B (на боль):** "Пока конкуренты ждут разработчика 2 месяца — ваш продукт уже работает"

> **Вариант C (на доверие):** "Создаю то, что сама использую каждый день — сайты, приложения и AI-агентов"

> **Вариант D (на специфику):** "От идеи до рабочего продукта — за 5 дней. Без технического брифа."

**Рекомендация:** Вариант A или D — конкретный срок сразу закрывает главную боль ("как долго").

## 4. Структура страницы

Порядок секций следует психологическому пути посетителя: от "что это?" до "хочу написать".

```
┌─────────────────────────────────────────────┐
│ 00. Navigation (sticky)                     │
├─────────────────────────────────────────────┤
│ 01. Hero — Кто + Для кого + Результат + CTA │  ← 5-секундный тест
├─────────────────────────────────────────────┤
│ 02. Marquee — бегущая строка навыков        │  ← разделитель + доп. инфо
├─────────────────────────────────────────────┤
│ 03. Кейсы / Портфолио                       │  ← главное доказательство
├─────────────────────────────────────────────┤
│ 04. Услуги                                  │  ← что конкретно нанять
├─────────────────────────────────────────────┤
│ 05. Это для вас / не для вас                │  ← фильтрация + доверие
├─────────────────────────────────────────────┤
│ 06. Процесс работы                          │  ← снять страх неизвестности
├─────────────────────────────────────────────┤
│ 07. О себе                                  │  ← личность после компетенции
├─────────────────────────────────────────────┤
│ 08. Отзывы (когда появятся)                 │  ← третья сторона
├─────────────────────────────────────────────┤
│ 09. FAQ                                     │  ← ответы на возражения
├─────────────────────────────────────────────┤
│ 10. Финальный CTA                           │  ← точка конверсии
├─────────────────────────────────────────────┤
│ Footer                                      │
└─────────────────────────────────────────────┘
```

**Почему именно этот порядок:**
- Hero отвечает "что это?" до решения о скролле
- Кейсы отвечают "умеет ли она?" до инвестиции во чтение
- Услуги отвечают "что нанять?" пока посетитель тёплый
- Процесс отвечает "каково это будет?" при переходе к рассмотрению
- Отзывы отвечают "можно ли доверять?" прямо перед CTA

## 5. Контент каждой секции

---

### 00. Navigation

**Назначение:** Ориентация, мгновенный доступ, сигнал профессионализма.

**Пункты меню (5 штук):**
```
Светлана Курчева [wordmark]   Кейсы   Услуги   Процесс   О себе   [Обсудить проект →]
```

**Требования:**
- Sticky — видима всегда
- CTA-кнопка визуально отличается от пунктов (градиентная кнопка, не ссылка)
- На мобильном: burger-меню или нижний nav-bar
- Эффект frosted glass при скролле: `backdrop-filter: blur(20px)`, тонкая нижняя рамка

---

### 01. Hero

**Назначение:** Ответить на 3 вопроса за 5 секунд — Кто? Что делает? Почему должно волновать?

**Структура блока:**

```
[Статус-бейдж]  ● Принимаю проекты

[Имя]           Светлана Курчева

[Заголовок]     Ваш сайт, приложение
                или AI-агент — за 3–7 дней

[Подзаголовок]  Создаю рабочие цифровые продукты для малого и среднего
                бизнеса с помощью AI-инструментов. Без долгих брифов,
                без лишних встреч — только задача и результат.

[CTA-кнопки]    [Расскажите о задаче →]    [Посмотреть кейсы ↓]

[Микротекст]    Отвечаю в течение 2 часов · Первая консультация бесплатна
```

**Статус-бейдж** (AI-brutalism тренд):
```
● Принимаю проекты · Апрель 2026
```
Пульсирующая зелёная/фиолетовая точка. Создаёт срочность + сигнал доступности.

**Правила hero-копирайтинга:**
- Заголовок: конкретный результат + срок, НЕ должность
- Подзаголовок: "для кого" + "как" + "что получат" в одном-двух предложениях
- Никаких абстракций: "инновационный", "профессиональный", "качественный" — запрещено
- Typewriter-анимация допустима как декоративный элемент, но НЕ как единственный носитель смысла

### 03. Кейсы / Портфолио

**Назначение:** Главное доказательство. Здесь решается — продолжать скроллить или уйти.

**Заголовок секции:** "Проекты"
**Подзаголовок:** "Задача — решение — результат"

**Структура каждого кейса (карточка):**
```
[Превью / скриншот]
[Тег категории]     Telegram-бот / Веб-приложение / AI-агент / Лендинг
[Название проекта]
[Клиент / контекст] — одна строка
[Ключевой результат] — выделен жирным: "время ответа: с 4 часов до 2 минут"
[Ссылка]            Подробнее →
```

**Количество кейсов:** 3–6 (меньше 3 = пусто, больше 8 = перегруз).

**Шаблон текста каждого кейса:**
```
ЗАДАЧА:    [Что хотел клиент — одно предложение]
РЕШЕНИЕ:   [Что было сделано — технология + подход]
РЕЗУЛЬТАТ: [Измеримый итог: время, деньги, конверсия, часы]
СРОК:      [X дней]
```

**Примеры кейсов для наполнения (структура):**

> **Кейс 1 — Telegram-бот для записи клиентов**
> Задача: Владелец сервиса тратил 3–4 часа в день на ответы в мессенджерах.
> Решение: Telegram-бот с интеграцией календаря и автоответами на типовые вопросы.
> Результат: 80% вопросов обрабатывает бот. Время ответа — с 4 часов до 2 минут.
> Срок: 6 дней.

> **Кейс 2 — Лендинг с формой заявок**
> Задача: Нужна страница для сбора заявок на вебинар без технической команды.
> Решение: Лендинг с формой, интеграция с Google Sheets и автоответами на email.
> Результат: 34 заявки в первую неделю после запуска.
> Срок: 3 дня.

> **Кейс 3 — AI-агент для обработки входящих запросов**
> Задача: Компания получала 50+ однотипных запросов в день, менеджеры выгорали.
> Решение: AI-агент на базе Claude API, обученный на FAQ и продуктовой базе.
> Результат: 70% запросов закрывается автоматически. Время на обработку сократилось с 2 минут до 8 секунд.
> Срок: 8 дней.

**CTA под кейсами:** "У меня похожая задача → обсудить"

### 05. Это для вас / Не для вас

**Назначение:** Отсеять нецелевых клиентов, повысить доверие через честность.

**Заголовок:** "Подходит ли вам работа со мной?"

```
✓ ЭТО ДЛЯ ВАС, ЕСЛИ:

  ✓ Есть конкретная задача или идея — хотите сделать быстро
  ✓ Нужен результат за дни, а не месяцы
  ✓ Хотите автоматизировать рутину без найма технической команды
  ✓ Нужен человек, который думает как предприниматель, а не только пишет код
  ✓ Открыты к рекомендациям: "А вам вообще нужен сайт, или хватит бота?"


✗ ЭТО НЕ ДЛЯ ВАС, ЕСЛИ:

  ✗ Нужна система на 100 000+ пользователей с нуля и highload-архитектура
  ✗ Хотите "всё и сразу" без чёткой задачи и готовы ждать год
  ✗ Нужна команда с project-менеджером, дизайнером и QA в штате
  ✗ Принципиально важен офис и личные встречи каждую неделю
```

Отрицательный список повышает доверие — показывает честность и самодостаточность.

### 06. Процесс работы

**Назначение:** Снять страх "что происходит после того, как я напишу?"

**Заголовок:** "Как мы работаем"
**Подзаголовок:** "4 шага от запроса до запуска"

```
Шаг 1: Разговор          │  Шаг 2: План             │  Шаг 3: Разработка      │  Шаг 4: Запуск
─────────────────────────│──────────────────────────│─────────────────────────│─────────────────
Вы описываете задачу.    │  Я предлагаю решение:    │  Вы получаете           │  Готовый продукт
Я задаю вопросы,         │  что делаем, сколько     │  промежуточные          │  опубликован.
чтобы понять суть.       │  стоит, сколько занимает.│  версии — можно         │  Передаю все
                         │  Вы соглашаетесь         │  давать обратную        │  доступы,
Формат: Telegram/Zoom    │  или корректируете.      │  связь по ходу.         │  объясняю
Время: 30–60 минут       │  Время: 24 часа          │  Время: 3–14 дней       │  как пользоваться.
Стоимость: бесплатно     │  после разговора         │  в зависимости от задачи│
```

**Примечание под схемой:**
"Не беру предоплату до согласования плана. Не называю срок, пока не понимаю задачу. Когда называю — держу."

Создаю сайты, приложения и AI-инструменты с 2024 года.

Специализируюсь на малом и среднем бизнесе: стартапах, сервисах,

### 08. Отзывы

**Назначение:** Third-party validation прямо перед финальным CTA.

> ⚠️ **Статус:** Блок добавляется после появления реальных отзывов.

**Структура каждого отзыва:**
```
[Фото клиента]
[Имя] — [Должность], [Компания]

"[Цитата с конкретным результатом: не "всё понравилось",
  а "получили бота за 4 дня, 80% заявок обрабатывается автоматически"]"

[Логотип компании / ссылка на проект]
```

**Требования:**
- Минимум 3, максимум 6 отзывов
- Формат: статическая сетка из 3 (не карусель)
- Обязательно: имя + фото + должность + компания
- Желательно: скриншот реального диалога или результата

**Пока нет отзывов — альтернативы:**
- Счётчики: "X проектов запущено · Средний срок Y дней · Z% клиентов возвращаются"

## 6. Стиль дизайна

### Настроение / Характер
**"Точный. Живой. Технологичный."**
Не корпоративный, не агентский. Живой голос конкретного человека с техническим бэкграундом. Дизайн демонстрирует мастерство без слов — сайт, созданный мастером, сам является доказательством.

### Цветовая палитра

```

Фон секций:          #0a0a0a  (чёрный)
Фон акцентный:       #0f0c29  (тёмно-фиолетовый для переходов)

Акцент 2 (розовый):     #ff6b8a
Градиент основной:      linear-gradient(135deg, #7c6fff, #ff6b8a)

Текст основной:       #ffffff
Текст вторичный:      rgba(255, 255, 255, 0.6)
Текст третичный:      rgba(255, 255, 255, 0.35)

```

**Почему эта палитра:** Фиолетово-розовый градиент — доминирующая эстетика AI-инструментов 2025–2026 (Midjourney, Runway, Pika, Claude). Биолюминесцентный эффект: тёмная база заставляет цвета светиться.

### Типографика

```

                  Gradient text: #7c6fff → #ff6b8a с drop-shadow

```

**Опциональный Google Font:** Inter (если добавить внешний шрифт) — для заголовков.

## 8. Анимации и интерактивность

### Принципы
1. Анимировать только `transform` и `opacity` — не `width/height/top/left`
2. Обязательно `@media (prefers-reduced-motion: reduce)` — отключает все анимации
3. `will-change: transform` только там, где реально нужно — убирать после анимации

### Scroll Reveal (обязательно)
```css
.reveal {
  opacity: 0;
  transform: translateY(30px);
  transition: opacity 0.7s cubic-bezier(0.16, 1, 0.3, 1),
              transform 0.7s cubic-bezier(0.16, 1, 0.3, 1);
}
.reveal.is-visible { opacity: 1; transform: translateY(0); }
```
IntersectionObserver с `threshold: 0.15`. Staggered delays для карточек: `.reveal-1` через `.reveal-4`.

### Анимации при загрузке (Hero)
- Имя: fade-in + slide-up, `animation-delay: 0.1s`
- Заголовок: fade-in + slide-up, `animation-delay: 0.2s`
- Подзаголовок: fade-in, `animation-delay: 0.35s`
- Кнопки: fade-in + scale, `animation-delay: 0.5s`
- Статус-бейдж: fade-in, `animation-delay: 0s` (первым)

### Marquee (бегущая строка)
```css
animation: marquee-scroll 30s linear infinite;
/* Пауза при hover */
mask-image: linear-gradient(to right, transparent 0%, black 10%, black 90%, transparent 100%);
```

### Hover на карточках кейсов
- `translateY(-4px)` + усиление glow
- Дополнительно: gradient border через `::before` с `opacity: 0 → 0.6`
- Опционально: 3D Tilt через mousemove (±10deg по X и Y)

### Gradient Text Sweep (заголовки секций)
```css
background-size: 200% auto;
animation: gradient-sweep 4s linear infinite;
```

### Aurora фон
```css
animation: aurora-drift-1 12s ease-in-out infinite alternate;
/* Второй орб: 15s */
```

### Page Loader (опционально)
Занавес при первой загрузке: `scaleY(1) → scaleY(0)` за 0.8s, с задержкой 1.5s.

## 9. Адаптивность и производительность

### Брейкпоинты
```
Desktop:  > 1200px  — полная сетка, горизонтальный процесс
Tablet:   860–1200px — 2 колонки для карточек
Mobile:   < 860px   — 1 колонка, вертикальный процесс
Mobile S: < 480px   — уменьшенные отступы, базовый font-size
```

### Мобильные приоритеты
- Floating CTA-кнопка внизу экрана
- Фото в hero — не съедает больше 40% экрана
- Touch targets: минимум 44×44px для всех интерактивных элементов
- Горизонтальный процесс → вертикальный на мобиле
- Marquee — меньше скорость (`animation-duration: 20s`)

### Производительность
- Изображения: WebP, сжатие, `loading="lazy"` для всего ниже fold
- Главная картинка (hero): eager loading, предзагрузка
- CSS: `will-change` только для активно анимируемых элементов
- Шрифты: системный стек (без внешних запросов) или `font-display: swap`
- Aurora фон: `position: fixed`, `z-index: -1`, не перерисовывается при скролле

### Целевые метрики
- LCP (Largest Contentful Paint): < 2.5s
- CLS (Cumulative Layout Shift): < 0.1
- Lighthouse Performance: > 85

## Ключевые принципы (не нарушать)

1. **Клиент-центричность.** Каждое предложение — тест: "Это про меня или про результат клиента?" Убирать всё, что только про автора.

2. **Конкретика > абстракции.** Ноль слов: "инновационный", "профессиональный", "качественный", "современный". Только: сроки, цифры, конкретные deliverables.

3. **Один доминирующий CTA на секцию.** Несколько конкурирующих — паралич решения.

4. **Доказательства до процесса.** Кейсы → Услуги → Процесс — именно в этом порядке.

5. **Сайт сам является портфолио.** Качество анимаций, внимание к деталям, производительность — всё это демонстрирует мастерство без слов.

6. **Честность как стратегия.** Прямые цены, реальные сроки, список "не для вас" — парадоксально повышают доверие.
