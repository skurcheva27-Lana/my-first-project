# Правила работы с проектом

## Структура файлов

- Каждая страница — один самодостаточный HTML-файл: HTML + CSS + JS внутри
- Никаких внешних библиотек, фреймворков и CDN
- Шрифты — только системные (`-apple-system, BlinkMacSystemFont, 'Segoe UI', sans-serif`)

## Дизайн

- **Фон:** тёмный (оттенки `#08080f` — `#302b63`)
- **Текст:** светлый (`#e2e0f0`, `rgba(255,255,255,0.6)` для второстепенного)
- **Акцентный цвет:** розово-фиолетовый (`#7c5cfc` / `#a78bfa` / `#ff6b8a`)
- Стиль карточек — glassmorphism: полупрозрачный фон + размытие + тонкая рамка
- Градиентный текст для заголовков
- Плавные анимации появления элементов

## Адаптивность

- Вёрстка адаптивна: хорошо выглядит на экранах от 320px до 1920px
- Использовать `clamp()` для размеров шрифтов
- Медиазапрос для мобильных: `@media (max-width: 600px)`
- На мобильных — увеличенные отступы для удобства касания

## Текст на страницах

- Простой и дружеский язык, без канцелярита и корпоративных штампов
- Короткие предложения, живые формулировки
- Обращение на «вы» (с маленькой буквы)

## Процесс работы

- Перед созданием нового файла — сначала описать, что планируется сделать, и дождаться подтверждения
- Перед редактированием существующего файла — прочитать его целиком
- Не добавлять функциональность, которую не просили
